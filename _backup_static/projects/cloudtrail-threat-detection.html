CloudTrail Threat Detection Platform

A cloud security product for detecting suspicious AWS activity in near real-time, prioritizing incidents, and accelerating analyst response.

GitHub Live Demo (Portfolio) Engineering Process

Problem

Security teams lacked clear visibility into high-risk identity actions across distributed AWS accounts and had slow triage cycles.

Research

Analyzed incident patterns, IAM abuse techniques, and analyst workflows to design detections tuned for practical alert quality.

Planning

Defined event ingestion, normalization, correlation, and scoring stages with explicit SLAs for latency and false-positive control.

Architecture

Event-driven pipeline using CloudTrail + EventBridge + Lambda for filtering, enrichment, and risk scoring before alert delivery.

Implementation

Created rule packs for suspicious API activity, impossible travel indicators, privilege escalation chains, and anomalous token use.

Security

Least-privilege IAM roles, encrypted transport, audit logging, and immutable evidence capture for post-incident investigations.

Challenges

Managing noisy events at scale and reducing alert fatigue without missing high-severity activity under bursty workloads.

Solutions

Context-aware suppression, weighted scoring, and staged rule confidence levels to maintain detection precision.

Performance

Designed for sub-minute alerting paths and scalable event throughput under account growth and regional expansion.

Lessons Learned

Detection quality depends as much on operational tuning and triage UX as on rule complexity alone.

Future Roadmap

Cross-account graph analysis, adaptive behavior baselines, and automated remediation playbooks.

Impact

Improved incident response readiness, shortened investigation loops, and increased confidence in cloud audit visibility.
